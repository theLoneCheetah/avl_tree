## 1. Выбор проекта
Проект выполняется на языке Python. Для выполнения работы в качестве модуля для тестирования был выбран учебный проект из курса алгоритмов и структур данных 1 курса. Проект реализует собственную структуру AVL-дерева - сбалансированного двоичного дерева поиска, для которого верно, что для каждой вершины высота поддеревьев отличается не более чем на 1.

Проект состоит из двух файлов:
* node.py - описывает класс узла дерева Node, хранящий ключевое значение value, дополнительную координату xcoord, ссылки на левое и правое поддерево и высоту поддеревьев
* avl_tree.py - содержит класс, реализующий статические методы для работы с AVL-деревом: функции вставки, удаления, выборки элементов.
Код исходного модуля расположен в папке bin/ проекта ([GitHub](https://github.com/theLoneCheetah/avl_tree/tree/main/bin)).

## 2. Анализ функциональности
Класс реализован статически и работает с передаваемым ему корневым узлом дерева, что позволяет не создавать экземпляр объекта и динамически проверять его узлы. Помимо методов вставки, удаления элементов, взятия среза, четырёх типов балансировки, он также содержит вспомогательные методы расчёта высоты, разницы высот и поиска минимального элемента левого поддерева. Метод вывода дерева использовался для отладки и для тестирования в данном случае плохо подходит.

Первоначально проект предназначался для решения задачи о поиске двух ближайших точек с помощью алгоритма заметающей прямой, поэтому координата xcoord для работы стандартного AVL-дерева не требуется. Однако исходный код модуля было решено оставить без изменений для демонстрации работы при тестировании.

Критически важные методы, подлежащие тестированию:
* Вставка элемента (метод insert) – должен корректно размещать узел в дереве и восстанавливать баланс после добавления.
* Удаление элемента (pop) – должен удалять узел с учётом трёх случаев (лист, один потомок, два потомка) и перебалансировать дерево.
* Балансировка – все четыре типа вращений (левое, правое, большое левое, большое правое) должны обеспечивать свойство AVL и возвращать ожидаемый корень.
* Получение среза (get_slice) – должен возвращать список пар (xcoord, value) для узлов, значение которых попадает в заданный промежуток включительно, упорядочив их по ключу value.

Также стоит проверить граничные случаи – пустое дерево, удаление несуществующего элемента, запрос среза вне диапазона значений и т.п. Эти сценарии покрывают основные сценарии использования и почти полностью гарантируют корректность структуры данных.

## 3. Написание тестов
Для тестирования используется встроенный модуль unittest. Тесты организованы в файле test_avl_tree.py и следуют подходу AAA (Arrange–Act–Assert) и принципам FIRST (Fast, Isolated, Repeatable, Self-validating, Timely).

Для тестирования ключевых методов и сценариев были написаны следующие методы unit-тестов:
* test_insert_single - проверяет вставку одного узла и корректность всех его полей.
* test_right_rotate - с помощью простой убывающей последовательности проверяет метод балансировки малым правым вращением (малое левое в модуле реализовано аналогично).
* test_big_left_rotate - вставкой трёх элементов проверяет балансировку большим левым вращением (большое правое в модуле реализовано аналогично).
* test_pop_node_with_two_children - создаёт дерево из 5 узлов и проверяет корректность удаления узла с двумя дочерними узлами и сбалансироваанность оставшегося дерева.
* test_get_slice_boundaries - проверяет получение среза с граничными значениями для дерева из 5 узлов.
* test_pop_empty - выполняет простую проверку попытки удалить узел из пустого дерева.

Для удобства в класс тестирования создан вспомогательный метод _is_avl, рекурсивно проходящийся по дереву и проверяющий правило сбалансированности.

## 4. Результаты запуска тестов и метрика code coverage
Для запуска файла тестирования напрямую можно использовать команду `python3 bin/test_avl_tree.py`.

Для запуска с помощью утилиты coverage и проверки покрытия модуля тестами можно использовать команду `python3 -m coverage run bin/test_avl_tree.py`. Так coverage запустится как оболочка для unittest и автоматически соберёт данные о покрытии модуля тестами (в файле .coveragerc указано исключение - сам файл тестов).

С помощью команды `python3 -m coverage report -m` можно вывести в консоль результаты работы метрики coverage. Скриншот результатов работы показан ниже.

Как видно из результатов unit-тестирования, все тесты были пройдены успешно. Файл avl_tree.py на 78% покрывается тестами (общий показатель 79%), что является достаточно хорошей цифрой. ## 5. Выводы

1. Были написаны 6 unit-тестов для тестирования работы модуля для реализации AVL-дерева. Тесты проверяют базовые публичные методы, корректность структуры после операций, выполнение свойства сбалансированности, граничные случаи.
2. Ключевые сценарии использования AVL-дерева учтены. Благодаря общему покрытию 79% можно утверждать, что наиболее важные участки кода протестированы и структура, скорее всего, соответствует правилам AVL-дерева.
3. В ходе тестирования обнаружен неиспользуемый кусок кода с 56 по 58 строку, который не влияет на работу структуры, но загрязняет модуль. Также присутствует метод вывода дерева, который для дальнейшей отладки может быть необязателен.
4. Unit-тестирование - один из наиболее полезных методов тестирования проектов, особенно на стадии разработки и первичной отладки, позволяющий выявить локальные недоработки и баги. Модуль unittest и метрика coverage обеспечивают автоматизированный запуск unit-тестов и подсчёт покрытия файлов написанными тестами, что упрощает их написание и поддержку.
